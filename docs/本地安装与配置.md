# 本地安装与配置

## 前置条件

需要一个兼容的 wxbot 项目，其目录包含：

```text
wxbot/
├── config/
├── core/
├── plugins/
└── utils/
```

插件使用 wxbot 当前 Python 环境，不单独启动进程。

## 自动安装

在插件仓库执行：

```powershell
.\install_local.ps1 -BotRoot "E:\path\to\your-bot"
```

脚本只覆盖插件 `main.py`。如果目标已经有 `config.json`，脚本会保留它。

## 手工安装

复制：

```text
plugin/wxsph_parser/main.py
```

到：

```text
YOUR_BOT_ROOT/wxbot/plugins/wxsph_parser/main.py
```

首次安装时，把：

```text
plugin/wxsph_parser/config.example.json
```

复制并重命名为：

```text
YOUR_BOT_ROOT/wxbot/plugins/wxsph_parser/config.json
```

## Finder 短链模式配置

```json
{
  "finder_bridge_enabled": true,
  "finder_bridge_base": "http://127.0.0.1:8790",
  "finder_bridge_api_key": "YOUR_FINDER_KEY",
  "finder_bridge_timeout": 30
}
```

检查 Finder 网关：

```powershell
curl.exe http://127.0.0.1:8790/health
```

必须达到：

```json
{
  "data": {
    "browser_started": true,
    "ready": true
  }
}
```

如果 wxbot 和 Finder 网关不在同一台机器，将 `finder_bridge_base` 改为 HTTPS
地址。不要使用公网明文 HTTP。

## sph 解析配置

```json
{
  "api_base": "http://127.0.0.1:8787",
  "api_key": "YOUR_WXSPH_KEY",
  "timeout": 18,
  "max_retries": 2
}
```

检查：

```powershell
curl.exe http://127.0.0.1:8787/health
```

`api_base` 指向 `wxsph-api`，不是 Finder 短链网关。两者端口和 API Key 可以
不同。

## 协议兜底配置

默认示例关闭：

```json
{
  "finder_protocol_enabled": false
}
```

只有 protocol-core 已提供兼容接口时才开启：

```json
{
  "finder_protocol_enabled": true,
  "finder_protocol_base": "http://127.0.0.1:9000/api",
  "finder_protocol_timeout": 15,
  "finder_protocol_cgi": 3906
}
```

protocol-core 必须能根据当前机器人 wxid 调用 Finder 详情 CGI，并返回完整媒体
URL 和可选解密 Key。

## 触发配置

推荐：

```json
{
  "auto_detect": false,
  "allow_group": true,
  "allow_private": true,
  "triggers": [
    "解析",
    "解析链接",
    "视频号解析"
  ]
}
```

`auto_detect=false` 表示只有显式触发才处理，降低误触发和接口调用频率。

当前插件还会忽略“没有引用上下文的普通分享消息”。推荐让用户引用卡片后发送
触发词。

## 回复配置

```json
{
  "send_summary": true,
  "send_video": true,
  "send_fallback_text": true,
  "video_max_size_mb": 300
}
```

说明：

- `send_summary`：发送标题、作者和原始 `sph`。
- `send_video`：下载后调用机器人视频发送接口。
- `send_fallback_text`：视频发送失败时返回链接或播放地址。
- `video_max_size_mb`：下载体积上限。

## CDN 候选配置

```json
{
  "try_compact_cdn": true,
  "try_full_cdn": true
}
```

插件会保留带签名参数的完整 URL，并可尝试压缩重复查询参数。不要手工拆分或重新
拼接 `token`、`sign`、`encfilekey`。

## 完整示例

参见：

```text
plugin/wxsph_parser/config.example.json
```

真实配置文件已被 `.gitignore` 排除。

## 更新

```powershell
git pull
.\install_local.ps1 -BotRoot "E:\path\to\your-bot"
```

更新后重启 wxbot。安装脚本不会覆盖现有 `config.json`。

## 卸载

停止 wxbot 后删除：

```text
YOUR_BOT_ROOT/wxbot/plugins/wxsph_parser/
```

删除前自行备份 `config.json`。不要把包含真实 Key 的备份放进 Git。
