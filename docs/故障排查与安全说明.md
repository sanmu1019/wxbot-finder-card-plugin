# 故障排查与安全说明

## 插件没有触发

检查：

1. `enabled` 是否为 `true`。
2. 群聊或私聊是否被 `allow_group/allow_private` 禁用。
3. 是否引用了视频号卡片。
4. 是否发送了配置中的触发词。
5. wxbot 是否在安装插件后重启。

默认会忽略没有引用上下文的普通分享消息。

## 提示缺少 objectId 或 objectNonceId

原因通常是机器人框架没有把原始引用 XML 放进：

```text
original_content
outer_content
content
```

不要根据 object ID 猜测 nonce。应检查机器人消息回调的原始结构。

排查日志时只打印 nonce 长度，不打印完整值。

## Finder 网关 401

检查：

```text
finder_bridge_api_key
```

是否与 Finder 网关 `FINDER_API_KEY` 一致。

## Finder 网关 503

检查：

```powershell
curl.exe http://127.0.0.1:8790/health
```

确认：

- Chromium 已启动。
- 视频号管理后台已扫码登录。
- 已进入内容管理页面。
- 已打开过作品分享菜单。
- `ready=true`。

## Finder 网关 502

可能原因：

- object ID 与 nonce 不属于同一张卡片。
- 管理后台页面接口发生变化。
- webpack 短链 service 未正确定位。
- 卡片已经失效。

## wxsph-api 401

检查插件：

```text
api_key
```

是否与解析器的 Key 一致。

## wxsph-api 500 或 502

常见原因：

- 元宝 Cookie 过期。
- 元宝接口返回结构变化。
- `sph` 已失效。
- 上游网络错误。

Finder 网关能生成 `sph`，不代表元宝解析一定成功。这是两个独立服务。

## protocol-core 失败

只有设置：

```json
{
  "finder_protocol_enabled": true
}
```

时才会走协议兜底。

检查：

- 当前机器人 wxid 是否可取得。
- `finder_protocol_base` 是否正确。
- `/Finder/GetCommentDetail` 是否存在。
- protocol-core 是否有当前机器人登录态。
- CGI 3906 是否能正常返回。
- 返回媒体是否包含完整 URL 或 `url + url_token`。

## 下载提示 URL 过期

卡片中的旧 CDN URL 可能包含完整参数，但签名仍然已经过期。

处理顺序：

1. 优先使用 Finder 网关生成新的 `sph`。
2. 或使用 protocol-core 刷新媒体 URL。
3. 不要继续重试旧 URL。

## 下载成功但不是视频

Finder 视频可能需要 `decode_key` 解密。协议接口必须返回与媒体 URL 同批次的
Key。

不要把不同请求的 URL、token 和 decodeKey 混用。

## 视频发送失败

插件会尝试：

```text
send_video_msg
-> 失败后 send_file_msg
```

检查：

- 文件大小是否超过机器人上传限制。
- `video_max_size_mb` 是否合理。
- wxbot 文件上传服务是否正常。
- 临时目录是否可写。

## 日志脱敏

安全日志可以包含：

```text
objectId
nonceLength
候选媒体数量
HTTP 状态码
错误类型
```

禁止包含：

```text
完整 objectNonceId
机器人 Cookie 和 session key
Finder API Key
元宝 Cookie
完整媒体 token
decodeKey
二维码
```

## Git 发布检查

提交前检查：

```powershell
git status --short
git check-ignore -v plugin\wxsph_parser\config.json
git diff --check
```

确认仓库中只有：

```text
config.example.json
```

没有真实 `config.json`。

## 调用频率

该插件涉及管理后台、元宝会话或微信协议私有接口。建议：

- 保持 `auto_detect=false`。
- 只允许明确触发。
- 不做高并发压力测试。
- 不批量枚举 object ID。
- 对群聊增加权限或频率限制。
